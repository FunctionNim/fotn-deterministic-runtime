# Visual Continuity Client Architecture

## Core Law

The runtime owns deterministic truth.
The visual client renders projection.

## Client Responsibilities

- render zones,
- render Function Box state,
- render replay timelines,
- render continuity dashboards,
- display battle-step interaction panels.

## Runtime Responsibilities

- authoritative sequencing,
- replay-safe resolution,
- visibility governance,
- synchronization,
- continuity persistence.

## Initial Visual Layers

1. Board View
2. Function Box Panel
3. Replay Viewer
4. Continuity Dashboard
5. Battle Resolution Overlay

## Long-Term Direction

- Unity runtime client
- replay theater mode
- continuity map rendering
- convergence observability layer
- multiplayer continuity presentation
