# Mode Interpreter Doctrine

Modes reinterpret pressure without rewriting deterministic truth.

Rules:

1. Modes may alter validation language.
2. Modes may alter pressure interpretation.
3. Modes may alter scoring meaning.
4. Modes must not bypass sequencing authority.
5. Modes must not bypass state transition authority.
6. Modes must remain replay-compatible.

Canonical mode interpretations:

- Standard: Function creates consequence.
- Bureaucracy: Approval creates reality.
- Instability: Consequence rewrites action interpretation.
- Rank: System evaluates function.
- Trial: System tests pressure survivability.

Purpose:

The ModeInterpreter lets one deterministic engine support multiple pressure experiences without corrupting replay truth.
