Create R15 — Replay Audit Fixture.

Goal:
Make replay audit behavior a first-class deterministic regression fixture.

Purpose:
R12 proved replay works. R13 made signatures canonical. R14 centralized scenarios. R15 should now prove that audit trails themselves are stable, inspectable, and regression-protected.

Current known state:

* R14 Scenario Registry is complete and CI-verified.
* Commit 8e3b2944 passed both GitHub Actions workflows.
* npm test currently passes 203/203.
* Registered scenarios include:
    * act-i:stone-room

    * act-i:stone-room:forced-failure
* RuntimeSignature is canonical and reusable.
* The repository is a TypeScript + C# runtime/library.
* Do not add a frontend web app.

Requirements:

1. Add a replay audit fixture module in TypeScript.
2. The fixture should read replay results through the scenario registry, not scattered direct imports.
3. The fixture should expose structured audit facts for each replay scenario.
4. Include fields such as:
    * scenarioId
    * auditEventCount
    * auditEventTypes or audit labels
    * firstAuditEvent
    * lastAuditEvent
    * auditHash
    * signatureCombinedHash
    * finalStateSummary
    * deterministicProof
5. Add tests proving:
    * same scenario → same audit facts

    * first and last audit events are stable

    * forced-failure scenario audit differs intentionally from normal Act I

6. Prefer structured assertions over console text.
7. Do not auto-update golden snapshots.
