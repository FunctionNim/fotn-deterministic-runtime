# Mode Equivalence Rules

Modes reinterpret runtime meaning without breaking deterministic truth.

Mode equivalence requires:

- identical sequencing,
- identical replay ordering,
- identical state hashes,
- identical pressure propagation.

Mode mismatch indicates:

- invalid interpretation drift,
- replay incompatibility,
- nondeterministic pressure reinterpretation,
- illegal mode-layer mutation.

Mode authority depends on deterministic simulation integrity.
